Detailed Description

Ordo high-level API.

This is the highest-level API for Ordo, which forgoes the use of cryptographic contexts completely, resulting in more concise code at the cost of reduced flexibility.

This function encrypts a buffer of a given length with the provided parameters.

Parameters:
inThis points to a plaintext buffer.
inlenThis contains the number of bytes of plaintext.
outThis points to the buffer to which to write the ciphertext.
outlenThis points to a variable which will contain the number of bytes written to out.
primitiveThis must point to the cryptographic primitive to be used.
modeThis must point to the cryptographic mode of operation to be used.
keyThis should point to a buffer containing the encryption key.
keySizeThis represents the length, in bytes, of the key buffer.
ivThis points to the initialization vector (this may be zero if the mode does not use an IV).
cipherParamsThis points to specific cipher parameters, set to zero for default behavior.
modeParamsThis points to specific mode of operation parameters, set to zero for default behavior.
Returns:
Returns ORDO_ESUCCESS on success, a negative error code on failure.
Remarks:
One downside of this function is that it is not possible to encrypt data in chunks - the whole plaintext must be available before encryption can begin. If your requirements make this unacceptable, you should use the encryption interface, located one level of abstraction lower - see encrypt.h.
The out buffer should have enough space to contain the entire ciphertext, which may be larger than the plaintext if a block mode with padding enabled is used. See remarks about padding in encrypt.h.
